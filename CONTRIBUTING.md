# Contributing

1. Open an issue describing the change.
2. Keep changes focused.
3. Add or update tests.
4. Use synthetic data only.
5. Do not submit third-party spreadsheet files, proprietary templates, credentials, or private repository references.
6. Run `python -m unittest discover -s tests -v` before opening a pull request.

New model families must include a model definition, required inputs, deterministic operations, fixtures, and tests.
